# Syntix 安装指南

> 支持 Chrome、Edge、Arc、Brave 等 Chromium 内核浏览器。

---

## 方式一：从 Chrome 网上应用店安装（推荐）

最简单的方式，一键安装，自动更新。

### 安装步骤

1. 打开 Chrome 浏览器
2. 访问 Chrome 网上应用店，搜索「**Syntix**」
3. 进入 Syntix 详情页，点击「**添加到 Chrome**」
4. 在弹出的确认框中点击「**添加扩展程序**」
5. 等待几秒钟，安装完成
6. 工具栏会出现 Syntix 图标

> Edge 浏览器同样支持从 Chrome 网上应用店安装，流程一致。

### 固定到工具栏（推荐）

安装完成后，图标可能被折叠在扩展菜单中：

1. 点击浏览器工具栏右侧的 **拼图图标**（扩展程序）
2. 在列表中找到 **Syntix**
3. 点击旁边的 **图钉按钮** 将其固定到工具栏

固定后，以后点击工具栏上的 Syntix 图标即可快速打开侧边栏。

---

## 方式二：离线安装（无法访问应用店时使用）

适用于无法访问 Chrome 网上应用店，或需要安装特定版本的情况。

### 第一步：下载并解压

1. 获取最新版本的插件压缩包（如 `syntix-chrome-v0.1.5.zip`）
2. 将压缩包解压到电脑上的**固定目录**
   - 建议放在文档或程序文件夹中，**不要放在桌面或下载文件夹**，避免误删
3. 解压后你会看到一个 `extension` 文件夹，里面包含 `manifest.json` 等文件
   - **请勿删除或移动该文件夹**，否则插件将无法正常工作

### 第二步：安装到浏览器

#### Chrome / Edge / Arc / Brave

1. 在地址栏输入以下地址并回车：
   - Chrome：`chrome://extensions/`
   - Edge：`edge://extensions/`
2. 开启 **「开发者模式」**（页面右上角或左下角的开关）
3. 点击 **「加载已解压的扩展程序」**（Edge 为「加载解压缩的扩展」）
4. 在文件选择器中，选择解压后的 **`extension`** 文件夹
5. 安装成功，工具栏出现 Syntix 图标

#### Firefox

1. 在地址栏输入：`about:debugging#/runtime/this-firefox`
2. 点击 **「临时载入附加组件…」**
3. 选择 `extension` 文件夹中的 **`manifest.json`** 文件
4. 安装完成

> 注意：Firefox 临时加载的扩展在浏览器重启后会失效。

---

## 安装后：登录开始使用

1. 点击工具栏上的 Syntix 图标，打开侧边栏面板
2. 输入你的账号和密码，点击「登录」
3. 登录成功后即可使用全部功能

详细操作请参考 [使用教程](./GUIDE.md)。

---

## 更新插件

### 从应用店安装的用户

插件会**自动更新**，无需手动操作。如需手动检查更新：

1. 打开 `chrome://extensions/`
2. 点击左上角的「**更新**」按钮

### 离线安装的用户

1. 下载新版压缩包，解压覆盖旧的 `extension` 文件夹
2. 打开扩展管理页面（`chrome://extensions/`）
3. 找到 Syntix，点击 **刷新按钮**（圆形箭头图标）
4. 更新完成，无需重启浏览器

---

## 卸载插件

1. 打开扩展管理页面（`chrome://extensions/`）
2. 找到 Syntix
3. 点击 **「移除」** 按钮
4. 确认移除

> 离线安装的用户卸载后可安全删除本地的 `extension` 文件夹。

---

## 常见问题

### Q：安装后看不到扩展图标？

点击工具栏的拼图图标（扩展程序），在列表中找到 Syntix，点击图钉按钮固定到工具栏。

### Q：提示「清单文件缺失或不可读取」？（离线安装）

请确保选择的是 `extension` 文件夹本身（而不是其父目录或子目录），该文件夹内应直接包含 `manifest.json` 文件。

### Q：浏览器提示「禁用开发者模式扩展程序」？（离线安装）

这是 Chrome 对开发者模式加载的扩展的正常安全提示，点击「**保留**」或「**取消**」即可继续使用，不影响功能。

### Q：扩展功能异常或没有反应？

1. 在扩展管理页面关闭再重新开启 Syntix
2. 刷新当前网页后重试
3. 如仍有问题，点击扩展管理页面 Syntix 下方的「错误」查看详情

### Q：离线安装的扩展每次打开浏览器都提示「禁用」？

这是 Chrome 的安全策略。如果你经常被提示，建议改用 Chrome 网上应用店安装，商店安装的扩展不会有此提示。

# Syntix 安装教程

> 适用版本：v0.1.2 | 支持 Chrome / Edge / Firefox / Arc 等主流浏览器

---

## 一、下载插件包

1. 下载最新版本的插件压缩包（如 `syntix-chrome-v0.1.2.zip`）
2. 将压缩包解压到电脑上的**任意固定目录**（建议不要放在桌面或临时文件夹，避免误删）

> 解压后你会看到一个 `extension` 文件夹，里面包含 `manifest.json`、`assets/`、`dist/` 等文件和目录。**请勿删除或移动该文件夹**。

---

## 二、安装到浏览器

### Chrome 浏览器

1. 打开 Chrome，在地址栏输入以下地址并回车：
   ```
   chrome://extensions/
   ```

2. 开启右上角的 **「开发者模式」**（拨动开关）

3. 点击左上角的 **「加载已解压的扩展程序」** 按钮

4. 在弹出的文件选择器中，选择解压后的 **`extension`** 文件夹

5. 安装成功后，工具栏会出现 Syntix 图标（蓝色 SY 标志）

---

### Edge 浏览器

1. 打开 Edge，在地址栏输入：
   ```
   edge://extensions/
   ```

2. 开启左下角的 **「开发人员模式」**

3. 点击 **「加载解压缩的扩展」**

4. 选择解压后的 **`extension`** 文件夹

5. 安装完成，扩展图标会出现在工具栏

---

### Firefox 浏览器

1. 打开 Firefox，在地址栏输入：
   ```
   about:debugging#/runtime/this-firefox
   ```

2. 点击 **「临时载入附加组件…」**

3. 选择解压后 `extension` 文件夹中的 **`manifest.json`** 文件

4. 安装成功后，扩展图标出现在工具栏

> 注意：Firefox 临时加载的扩展在浏览器重启后会失效。如需持久使用，建议通过 Firefox Add-ons 商店安装。

---

### Arc / Brave / Opera 等 Chromium 内核浏览器

安装步骤与 Chrome 基本一致：
1. 打开扩展管理页面
2. 开启开发者模式
3. 加载已解压的扩展，选择 `extension` 文件夹

---

## 三、固定扩展图标（推荐）

安装后扩展图标可能被折叠在扩展菜单中，建议将其固定到工具栏：

1. 点击工具栏右侧的 **拼图图标**（扩展程序）
2. 找到 **Syntix**
3. 点击右侧的 **📌 图钉按钮** 将其固定

---

## 四、开始使用

### 登录账号

1. 点击工具栏上的 Syntix 图标
2. 在弹出窗口中输入账号密码登录
3. 登录后即可使用全部功能

### 发布文章

1. 打开任意文章页面（博客、新闻、技术文章等）
2. 点击浏览器侧边栏中的 Syntix 面板
3. 选择目标发布平台（支持知乎、今日头条、CSDN、微信公众号等 22 个平台）
4. 系统会自动提取文章内容并跳转发布

### 操作方式

1. **侧边栏面板**：点击浏览器侧边栏中的 Syntix 图标，打开操作面板
2. **页面按钮**：在支持的文章页面上，右上角会自动出现发布按钮
3. **弹窗**：点击工具栏的 Syntix 图标可快速访问

---

## 五、更新插件

当有新版本发布时：

1. 下载新版压缩包并解压（覆盖旧的 `extension` 文件夹，或解压到新位置）
2. 打开扩展管理页面（`chrome://extensions/`）
3. 找到 Syntix，点击 **刷新按钮**（圆形箭头图标）
4. 更新完成，无需重启浏览器

---

## 六、卸载插件

1. 打开扩展管理页面（`chrome://extensions/`）
2. 找到 Syntix
3. 点击 **「移除」** 按钮
4. 确认移除即可

> 卸载后可安全删除本地的 `extension` 文件夹。

---

## 常见问题

### Q: 安装后看不到扩展图标？
检查是否开启了「开发者模式」，并确保选择了正确的 `extension` 文件夹（里面应该有 `manifest.json` 文件）。

### Q: 提示「清单文件缺失或不可读取」？
请确保你选择的是 `extension` 文件夹（而不是其父目录或子目录），该文件夹内应包含 `manifest.json`。

### Q: 扩展功能异常或页面没有反应？
1. 在扩展管理页面关闭再重新开启 Syntix
2. 刷新当前网页后重试
3. 如仍有问题，点击扩展管理页面的「错误」查看详细信息

### Q: 浏览器提示「禁用开发者模式扩展程序」？
这是 Chrome 的正常提示，点击「取消」即可继续使用。通过开发者模式加载的扩展会有此提示，不影响功能。
